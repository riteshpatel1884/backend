Why do we use sometime run and sometime without run in script command?
Ex: npm start or npm test and npm run dev or npm run concurrent

Ans: These script names have built-in shortcuts. They are equivalent to
- npm run start
- npm run test
- npm run stop
- npm run restart

Later, developers started creating many custom scripts (dev, build, lint, etc.), so npm
introduced the generic form: npm run <script-name>
